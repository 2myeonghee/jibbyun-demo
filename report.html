03
AI 엔진 설계 권장안
💡 헬미닥(HelpMeDoc) 협력 사례처럼, 핵심은 도메인 특화 시스템 프롬프트 + 검증된 사례 RAG입니다. Claude API 사용 시 공동주택관리법 전문 지식을 캐시 가능한 시스템 프롬프트로 구성하면 비용을 최대 90% 절감할 수 있습니다.

🧠 추천 AI 스택

  • 기본 모델: Claude Sonnet 4.5 (응답 품질·속도·비용 균형)
  • 고난도 법률 해석: Claude Opus 4 (명확한 법조문 해석 필요 시)
  • 빠른 분류/요약: Claude Haiku 4 (의도 분류, 요약 처리)
  • 벡터 DB: Supabase pgvector (아파트너 기존 인프라 활용 가능)
  • 임베딩: text-embedding-3-small 또는 Claude embedding

📋 시스템 프롬프트 구조

  • 역할 정의: "공동주택 법률 전문 AI 상담사"
  • 핵심 지식: 공동주택관리법 전문 + 시행령 캐시
  • 응답 형식: 구조화된 단계별 가이드
  • 면책 조항: 모든 응답에 자동 포함
  • 에스컬레이션: "전문가 필요" 조건 명시
  • 언어 톤: 친근하되 전문적, 판단 자제

🗄️ RAG 데이터 수집 우선순위

  • 공동주택관리법 전문 + 시행령·시행규칙
  • 국토교통부 유권해석 사례 (최근 5년)
  • 법원 판례 (입대의 관련, 200건+)
  • 중앙공동주택관리지원센터 상담 사례
  • 아파트너 자체 상담 이력 (익명화)
  • 황해 법무법인 처리 사례 (익명화)
  • 아파트아이·피터팬 커뮤니티 FAQ
  • 관리사무소 업무 매뉴얼 (국토부 배포)
04
UX 개선 포인트 (실사용 기반)

🔴 현재 UX 문제점

  • 해임 외 선택지 선택 시 다시 해임으로 유도 — 사용자 이탈 유발
  • 오른쪽 패널이 결과 전까지 비어 있어 첫인상 허전함
  • 모바일에서 3컬럼 레이아웃이 겹쳐 보임
  • 법령 클릭 시 alert() 사용 — 브라우저 네이티브라 조잡
  • 타이핑 인디케이터가 사라지기 전 스크롤이 튀는 현상
  • 입력창 placeholder 텍스트가 목업과 다름 ("키피라이팅 대충")

🟢 개선 방향

  • 시작 화면에 "오늘 자주 묻는 질문 3가지" 표시로 첫인상 개선
  • 오른쪽 패널: 대화 시작 전부터 "자주 묻는 사례" 미리 표시
  • 모바일: 하단 드로어로 오른쪽 패널 전환
  • 법령·사례 상세: 인라인 슬라이드 아웃 패널 사용
  • 대화 중간 저장·이어가기 기능 (로컬스토리지 최소 활용)
  • 피드백 버튼 (👍/👎) 각 AI 응답 하단에 추가
05
법적 리스크 및 대응 방안
리스크세부 내용대응 방안
법률 자문 vs 법률 정보 AI가 구체적 행동을 권고하면 변호사법 위반 소지 모든 응답에 "법률 자문이 아닌 일반 정보" 고지. 결정적 조언은 "전문가 상담 권유"로 종결
개인정보 처리 단지명·입주자 정보 입력 시 개인정보보호법 적용 단지명 등 식별 정보 비저장 원칙. 암호화 처리. 개인정보처리방침 명시
AI 오류 책임 잘못된 법령 해석으로 사용자가 피해를 입을 경우 이용약관에 AI 오류 면책 조항 포함. 중요 안건은 반드시 전문가 검토 권유
판례 저작권 법원 판례 전문 활용 시 저작권 문제 가능성 국가법령정보센터 공개 판례 활용 (공공저작물). 원문 링크 제공 방식 권장

권장 개발 로드맵
Sprint 1
1~2주

기반 인프라 구축

  • 백엔드 API 서버 구축 (FastAPI or Next.js API Routes)
  • Claude API 연동 + 기본 시스템 프롬프트 작성
  • 법적 면책 조항 및 이용약관 적용
  • 기본 사용자 인증 (아파트너 SSO 연동)
Sprint 2
3~4주

판례 DB + RAG 구축

  • 공동주택 판례 200건 수집·정제·임베딩
  • Supabase pgvector 세팅 + 유사 사례 검색 API
  • 공동주택관리법 전문 시스템 프롬프트 완성
  • 해임 시나리오 실 AI 응답으로 전환
Sprint 3
5~6주

문서 생성 + 시나리오 확장

  • 해임 동의서·안내문 자동 생성 (DOCX 출력)
  • 관리비 분쟁, 층간소음 등 5개 시나리오 추가
  • 대화 내역 저장·불러오기 (Supabase)
  • 사용자 피드백 수집 (👍/👎)
Sprint 4
7~8주

모바일 최적화 + 전문가 연결

  • 모바일 반응형 완성 (하단 드로어 패널)
  • 황해 법무법인 전화/카카오 연결 CTA
  • 안전성 테스트 + 베타 단지 파일럿
  • 사용 로그 분석 → 프롬프트 튜닝 1차

결론: 이 데모의 의미와 다음 단계

이번 데모는 UI/UX 검증과 시나리오 흐름 확인을 목적으로 제작되었습니다.

가장 중요한 다음 단계는 ① 실제 AI 엔진 연동과 ② 공동주택 전문 판례 DB 구축입니다. 이 두 가지가 완성되면 현재 데모를 즉시 실서비스 수준으로 전환할 수 있습니다.

황해 법무법인과의 협력을 통해 판례·사례 데이터 확보를, 아파트너 인프라를 통해 인증·단지 연동을 빠르게 해결할 수 있습니다.
